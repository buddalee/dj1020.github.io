Hexo 發佈到 GitHub 自訂網域(網址)的方式，設 CNAME

By 閃亮亮dj

發表於 2015-09-28

假設你已經買好 GoDaddy 的網域，而且設定好 CNAME

簡單提一下設定 CNAME 的流程，如果你買的網域叫 super383838.com
登入 GoDaddy，
在你的網域按 Manage 按鈕，
到 DNS Zone File，
然後 Add Record，
選 CNAME，
Host 欄填 blog 的話，你的部落格的網址就會是 http://blog.super383838.com
Points To 欄填 {username}.github.io 把 {username} 換成你的 github 帳號，
完成後記得 Save Changes。

此後，連到 http://blog.super383838.com 就跟連到 http://{username}.github.io 是一樣的了。

設定 GitHub CNAME 踩雷心得

我按照 GitHub 官網指示，加了一個叫 CNAME 的檔案(無副檔名)在 master branch 的根目錄裡，檔案內容只有一行：

也就是我想要的部落格網域，所以別人就可以連到 http://blog.dj1020.net 看到我的部落格。

加完這個檔案，一切順利正常，http://blog.dj1020.net 馬上可以正常顯示，而且如果連到原本的 http://{username}.github.io 要看部落格，還會被 GitHub 自動導向 http://blog.dj1020.net 這樣很好！

問題在於，CNAME 這個檔案是由我手動在 GitHub 上產生，但是 Hexo 並不知道，所以一旦 Hexo deploy 之後，CNAME 就被蓋掉了。所以再打開 http://blog.dj1020.net 就出現這樣的畫面：

解法：安裝 hexo-generator-cname

這個外掛會幫你產生 CNAME 檔，並且在 deploy 時一同 push 到 GitHub 上。安裝也很簡單，不用任何設定，我想它是去抓 _config.yml 裡的 url 設定值。

  1. $ npm install hexo-generator-cname --save
  2. 重新 deploy $ hexo deploy -g
  3. Done
