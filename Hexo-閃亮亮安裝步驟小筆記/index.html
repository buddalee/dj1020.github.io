Hexo 3.0 發佈，閃亮亮安裝步驟小筆記

By 閃亮亮dj

發表於 2015-09-23

從無到有 Hexo 3.0 在 GitHub 發佈流程，還有踩雷心得和解法

因為平常就有用 npm 和 git 所以這兩部份的安裝流程沒在裡面

  1. 把 npm 和 git 裝好

    • brew install npm
    • git (Mac 內建)

  2. $ npm install -g hexo-cli

  3. $ hexo init hexo

  4. $ cd hexo

  5. $ npm install

  6. 設定 _config.yml

  7. 下載 jacman 佈景主題

    • $ git clone https://github.com/wuchong/jacman.git themes/jacman
    • 把 _config.yml 的 themes 設成 jacman

  8. 建立這一篇文章

    • $ hexo new "Hexo 閃亮亮安裝步驟小筆記"

  9. 編輯 hexo/source/_posts/Hexo-閃亮亮安裝步驟小筆記.md

  10. 產生網頁 $ hexo generate

  11. 打開 server 看一下效果 $ hexo server，開 http://0.0.0.0:4000/

  12. 試試看 deploy $ hexo deploy -g (部署網站前先產生靜態檔案)

  13. 發現沒效，原來是要在 _config.yml 設 deploy 參數，參考這一篇: 使用hexo以及github page建立自己的部落格

    【重要】 閃亮亮先去 github 建一個 repo，但因為建錯 repo 名字，所以沒有頁面可以看……，記得建 repo 的時候一定要用 username.github.io 才會被 github 當成一個靜態頁網站。

  14. 再 deploy 一次試試看 $ hexo deploy -g

  15. 結果出現：

  16. 來去 google 找解，找到：

    1) Hexo Deployer not found: github
    2) 搭建 hexo，在執行 hexo deploy 後,出現 error deployer not found:github 的錯誤
    3) Hexo deploy 官方文件

    Hexo 3.0 開始 type 要改成 git，還有 repository 換成 repo，而且聽說新的 repo 是不能 deploy 上去的，要先自己 push 一個 commit，但說真的我不知道一開始 commit 要推什麼上去，先這樣好了：

  17. 再試一次 deploy $ hexo deploy -g，還是

  18. 不死心，再查 google 大神

    1) ERROR Deployer not found: github #1040

    說要手動在 hexo 專案資料夾下安裝 npm install hexo-deployer-git --save

  19. 裝完再試一次 $ hexo deploy -g，成功！推上去了，但是要去哪裡看我的站呢？

  20. 慘劇，原來建 repo 的時候一定要用 username.github.io 才會被當成一個 page，那就重建 repo 吧

    引用： 如何搭建一個獨立博客——簡明Github Pages與Hexo教程

    使用GitHub Pages建立博客與GitHub建立好鏈接之後，就可以方便的使用它提供的Pages服務，GitHub Pages分兩種，一種是你的GitHub用戶名建立的 username.github.io 這樣的用戶&組織頁（站），另一種是依附項目的pages。
    想建立個人博客是用的第一種，形如cnfeat.github.io這樣的可訪問的站，每個用戶名下面只能建立一個。

  21. 去 github 發現不用重建，改 repo 名字就好了，然後再重新 deploy 一次 $ hexo deploy -g

  22. 嘿嘿！終於成功了吧~ 閃亮亮大師之路 開張，底下那個 theme 作者的頭像要改一下。
